# AGENTS.md

## 結論（先に要点）

- このファイルは、エージェントが bonsAI を安全かつ一貫して変更するための行動ガイド。
- 変更時は「軽量・稼働継続性・SSE体感品質」を最優先にする。
- 実装後は必ずビルド/テスト/起動確認を実行し、失敗を残したまま終了しない。

## プロジェクト概要（1-3行）

Raspberry Pi 5 (8GB) 単体で、センサー連携のローカルLLMチャットを提供する。
主要構成は `bonsAI_front` (Preact UI), `bonsAI_server` (Go BFF), `bonsAI_LLM` (llama.cpp運用補助)。
通信は localhost 前提、応答は SSE ストリーミングを標準とする。

## システム前提

- ハードウェア: Raspberry Pi 5 (8GB必須)
- OS: Raspberry Pi OS 64-bit
- 通信: localhost (127.0.0.1) 上のHTTP/JSON
- 稼働方式: Docker Compose または systemd

## 作業前チェック（必須）

1. 変更対象ディレクトリの構成と既存コードを確認する。
2. 影響範囲を特定し、SSE経路とフォールバック動作への影響有無を明示する。
3. 新規依存の追加時は、メモリ使用量と起動時間への影響を説明する。

## セットアップ/ビルド/テスト

リポジトリルートで実行すること。

- 開発一括起動: `make dev-up`
- 停止: `make dev-down`
- 状態確認: `make dev-status`
- Docker起動: `make docker-up`
- Docker停止: `make docker-down`
- Dockerログ: `make docker-logs`

変更時の推奨チェック:

- Frontendビルド: `cd bonsAI_front && bun run build`
- Backendテスト: `cd bonsAI_server && go test ./...`

## 実装ルール（重要）

- 軽量性優先: 常駐プロセス/メモリ消費を増やす変更は最小化する。
- 局所変更優先: 関連しないリファクタや命名変更を混ぜない。
- 可観測性: 障害時に原因が追えるログ/エラーメッセージを維持する。
- 並行処理: Go BFFでは goroutine を活用し、待ち時間を最小化する。
- フェイルソフト: センサー未取得や保存失敗で会話本体を止めない。
- SSE維持: トークン中継を途切れさせない。同期的な重処理を応答経路に置かない。

## コーディング規約（要点）

- Frontend: Preact + Vite 構成を維持し、重い依存を安易に追加しない。
- Backend: ドメイン/ユースケース分離を崩さない（`internal/domain`, `internal/usecase`）。
- 設定値: localhost 前提を崩す変更は README と設定を同時更新する。
- エラー処理: 外部I/O（LLM/Sensor/保存）ではタイムアウトと失敗時の挙動を明示する。

## ガードレール / 禁止事項

- 秘密情報（APIキー、トークン、秘密鍵）をコード・ログ・設定に埋め込まない。
- 根拠のないモデル変更（サイズ/量子化/コンテキスト長）を行わない。
- 稼働継続性を下げる破壊的変更を、検証手順なしでマージしない。
- 会話完了後保存を同期処理へ戻さない（レイテンシ悪化を防ぐ）。

## 受入基準（変更ごとに満たす）

1. 変更範囲に応じたビルド/テストが通る。
2. SSEストリーミング応答が途切れない。
3. センサー取得失敗時も会話機能が継続する。
4. 保存失敗が応答処理へ波及しない。
5. 変更点と検証結果が説明可能である。
